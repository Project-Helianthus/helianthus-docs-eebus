# Vaillant VR940f / myVaillant eeBUS Target

## Purpose

VR940f/myVaillant is the first Helianthus eeBUS lab target. The first milestone
is raw visibility: sessions, services, topology, pairing state, and raw
evidence through read-only MCP.

## Non-Goals

- No GraphQL exposure.
- No Portal consumer workflow.
- No Home Assistant entities.
- No command routing.
- No promoted semantic fields.

## Evidence To Capture

- runtime status;
- local and remote identity with redaction;
- pairing state;
- sessions;
- visible and paired services;
- entity and feature graph;
- usecase claims;
- degraded states;
- restart and reconnect behavior.

## Transport Role Observation

`EV-20260720-001` is a redacted partial/negative observation for protected
outbound protocol pairing. It observed the local first-trust registration transition
with automatic acceptance disabled, but did not observe a completed candidate,
SHIP session, OOB confirmation, durable trust write, or trusted reconnect.
Accordingly, this page records VR940f only as the intended private outbound
pairing target for that attempt; it does not establish a reachable endpoint,
initiator/responder role, stable protocol-service identity, transport completion, or any
device semantic. Endpoint and identity values remain private and are excluded
from this page and all public outputs.
